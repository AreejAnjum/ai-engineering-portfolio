# Day 4 - VS Code Workflow

## Goal

Today I configured VS Code as my main AI Engineering workspace. I added workspace settings, recommended extensions, formatting rules, and terminal setup.

## What I learned today

1. VS Code is my main editor for writing code, documentation, and project notes.
2. Workspace settings help keep the project consistent.
3. `.vscode/settings.json` stores project-specific editor settings.
4. `.vscode/extensions.json` recommends useful extensions for this project.
5. Format on save helps keep code clean automatically.
6. The integrated terminal lets me run Git and project commands without leaving VS Code.
7. Good editor setup makes daily development faster and more professional.

## VS Code settings added

```json
{
  "editor.formatOnSave": true,
  "editor.wordWrap": "on",
  "editor.tabSize": 2,
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  "explorer.confirmDelete": true,
  "terminal.integrated.defaultProfile.osx": "zsh",
  "markdown.preview.breaks": true
}
```
