# Day 11 - Accessibility

## Goal

Today I improved the accessibility of my Personal Learning Dashboard. The goal was to make the page easier to use with keyboard navigation, labels, helper text, and clear focus states.

## What I learned today

1. Accessibility means making a website easier for more people to use.
2. Keyboard navigation is important because not every user uses a mouse.
3. A skip link helps keyboard users jump directly to the main content.
4. Labels should be connected to inputs using matching `for` and `id` values.
5. Focus styles show which element is currently selected.

## Key concepts

| Concept             | Simple meaning                        |
| ------------------- | ------------------------------------- |
| Accessibility       | Making a website easier for all users |
| Keyboard navigation | Using `Tab` instead of a mouse        |
| Skip link           | Link to jump directly to main content |
| Label               | Text connected to an input            |
| Focus state         | Visual sign showing selected element  |
| `aria-describedby`  | Connects helper text to a form field  |
| Required field      | A field the user must fill            |
