# Day 3 - Git Basics

## Goal

Today I learned the basic Git workflow: check changes, add changes, commit changes, and push changes to GitHub.

## What I learned today

1. Git is used to track changes in my code and documentation.
2. GitHub is used to store my project online.
3. `git status` shows which files changed.
4. `git add .` prepares all changed files for commit.
5. `git commit -m "message"` saves a snapshot of my work.
6. `git push` uploads my local commits to GitHub.
7. A good commit message should be short, clear, and meaningful.

## Commands I practiced

```bash
cd ~/Developer/ai-engineering-portfolio
git status
git add .
git commit -m "docs: add day 3 git basics notes"
git push
```

## Important command meanings

| Command                   | Meaning                            |
| ------------------------- | ---------------------------------- |
| `git status`              | Shows changed files                |
| `git add .`               | Stages all changes                 |
| `git commit -m "message"` | Saves a snapshot with a message    |
| `git push`                | Uploads commits to GitHub          |
| `git log --oneline`       | Shows commit history in short form |
