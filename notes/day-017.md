# Day 17 - JavaScript Validation

## Goal

Today I improved the JavaScript task rendering feature by adding basic validation. The goal was to handle invalid task data before showing it on the dashboard.

## What I learned today

1. Validation means checking data before using it.
2. A function can keep logic reusable and clean.
3. `filter()` creates a new array with only valid items.
4. `includes()` checks if a value exists inside an array.
5. Invalid data should be handled safely instead of breaking the UI.

## Key concepts

| Concept        | Simple meaning                         |
| -------------- | -------------------------------------- |
| Validation     | Checking data before using it          |
| Function       | Reusable block of logic                |
| `filter()`     | Keeps only items that pass a condition |
| `includes()`   | Checks if a value exists in a list     |
| Invalid data   | Data that is missing or not allowed    |
| Safe rendering | Showing only clean and usable data     |

## What was difficult?

## How I solved it

I learned that real applications often receive imperfect data from users, APIs, databases, or AI systems. Validation protects the UI from broken or unsafe data.
