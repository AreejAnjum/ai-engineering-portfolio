# Day 23 - TypeScript-Style Data Contracts

## Goal

Today I improved my JavaScript code with TypeScript-style data contracts using `// @ts-check` and JSDoc comments. The goal was to make the task data structure clearer and safer without changing the project into full TypeScript yet.

## What I learned today

1. TypeScript helps catch mistakes earlier by describing what data should look like.
2. JSDoc can add type-like information inside normal JavaScript.
3. `// @ts-check` asks VS Code to check JavaScript more carefully.
4. A task object should follow a clear structure.
5. Cleaner data contracts make code easier to read, debug, and maintain.

## Key concepts

| `// @ts-check` | Tells VS Code to check JavaScript more strictly       |
