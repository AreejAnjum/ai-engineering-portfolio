# Day 24 - Validation Test Cases

## Goal

Today I added simple validation test cases for my JavaScript task data. The goal was to check that valid tasks pass and invalid tasks fail before being rendered on the dashboard.

## What I learned today

1. Validation tests help prove that data rules are working.
2. A valid task should follow the expected task structure.
3. Invalid data should be detected before it reaches the UI.
4. `console.info()` can help inspect test results during learning.
5. Clean code should handle both correct and incorrect data.

## Key concepts

| Concept          | Simple meaning                                            |
| ---------------- | --------------------------------------------------------- |
| Validation test  | A check that proves rules are working                     |
| Valid task       | A task with correct fields and allowed values             |
| Invalid task     | A task with missing or wrong data                         |
| `console.info()` | Shows useful developer information in the browser console |
| Type warning     | A signal that data does not match the expected type       |
| Edge case        | A situation where data may break the app                  |

## What was difficult?

Understanding why I should test invalid data on purpose.

## How I solved it

I learned that real apps do not only receive perfect input.

```text
Good data → should pass
Bad data → should fail safely
```

This makes the dashboard more reliable.
