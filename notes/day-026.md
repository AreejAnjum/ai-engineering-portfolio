# Day 26 - Quality Day

## Goal

Today I improved the quality of my Personal Learning Dashboard by making form error handling clearer, focusing the correct invalid field, adding `maxlength` protection, and documenting manual quality checks.

## What I learned today

1. Quality means checking both success and failure paths.
2. Error messages should help the user fix the problem.
3. `maxlength` prevents users from typing too much text.
4. `aria-invalid` helps communicate invalid fields.
5. A quality checklist makes manual testing more reliable.

## Key concepts

| Concept        | Simple meaning                             |
| -------------- | ------------------------------------------ |
| Quality check  | A test to confirm the feature works well   |
| Failure path   | What happens when the user makes a mistake |
| `maxlength`    | Limits how much text can be typed          |
| `aria-invalid` | Marks a field as invalid for accessibility |
| `role="alert"` | Announces important error messages         |
| Manual testing | Testing the app by using it yourself       |

## What was difficult?

Understanding why small form improvements matter after the feature already works.

## How I solved it

I learned that professional features should not only work when everything is correct. They should also guide users when something is wrong.

```text
Valid input → task added
Invalid input → clear error + correct field focused
```
